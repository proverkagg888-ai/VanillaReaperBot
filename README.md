# VanillaReaperBot
Простой Telegram-бот с модерацией и развлекательными командами (все данные в памяти).

Файлы в репозитории:
- `main.py` — основной файл бота (готов к запуску)
- `requirements.txt` — зависимости
- `Dockerfile` — контейнер для запуска
- `.env.example` — пример переменных окружения

Требования
- Python 3.11+
- Установить зависимости:

```bash
pip install -r requirements.txt
```

Переменные окружения
- `BOT_TOKEN` — токен Telegram-бота. Вы можете положить его в переменную окружения или в файл `.env` (если используете `python-dotenv`). См. `.env.example`.

Локальный запуск

```bash
# экспорт переменной
export BOT_TOKEN="<токен_бота>"
# запустить
python main.py
```

Запуск в Docker

```bash
# сборка
docker build -t vanillareaperbot:latest .
# запуск (передать токен как env)
docker run -e BOT_TOKEN="$BOT_TOKEN" vanillareaperbot:latest
```

Деплой на bothost.ru

- Загрузите образ или используйте Dockerfile в настройках приложения на bothost (если платформа поддерживает билд из репозитория).
- В настройках приложения укажите переменную окружения `BOT_TOKEN`.
- Запуск обычно происходит автоматически после билда; убедитесь, что `CMD` остаётся `python main.py`.

Примечания
- Если во время сборки вы видите предупреждение про запуск `pip` как `root` — это нормально для контейнера, но можно создать непользовательского пользователя (в Dockerfile это сделано).
- Если хотите, могу добавить `Procfile` или настроить CI/CD для автоматического деплоя.
